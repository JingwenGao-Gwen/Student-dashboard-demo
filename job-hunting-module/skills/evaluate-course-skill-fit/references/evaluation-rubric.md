# Course-Skill Evaluation Rubric

## Evidence hierarchy

Use evidence in this order:

1. Learning outcomes explicitly state students will perform the skill.
2. Syllabus topics explicitly teach the skill.
3. Projects, laboratories, assignments, or assessments require the skill.
4. Course title explicitly names the skill and the syllabus confirms it.
5. Course description explains substantive instruction.

Prerequisites are exclusion/context evidence, not proof that the course teaches the skill.

## Direct training

Choose `direct_training` when at least one strong condition holds:

- The skill is an explicit learning outcome.
- Multiple syllabus topics teach the skill.
- A project, lab, assignment, or assessment requires students to use the skill.
- A tool or programming language is explicitly taught or used as a central course method.

Do not choose direct training merely because a term appears once.

## Foundational support

Choose `foundational_support` only when:

- The taught concept is a recognized prerequisite for the target skill.
- The course materially develops that concept.
- The evaluator can explain a concrete causal bridge in one sentence.

Example: probability and statistical inference can support A/B testing because they teach hypothesis tests, confidence intervals, and uncertainty estimation.

Do not use foundational support to mean “generally useful.”

## Not related

Choose `not_related` for:

- Ambiguous senses: `survey course`, `literature survey`, `market` in an unrelated meaning.
- Prerequisite-only mentions.
- Generic words such as analysis, design, management, communication, research, or model without skill-specific context.
- A course in the same industry that does not teach the target method.
- General cognitive or personality benefits.
- Evidence that only describes instructor research or background.

## High-risk skills

Apply extra scrutiny to:

- 用户研究 / user research
- 市场调研 / market research
- 需求分析 / requirements analysis
- 项目管理 / project management
- 数据分析 / data analysis
- 文案写作 / copywriting
- 实验设计 / experimental design
- A/B Test

For these skills, require method-specific evidence rather than generic words.

## Semantic counterexamples

- `This course is designed as a survey of monetary theories` → not user research.
- `Students conduct questionnaire design and user interviews` → direct user research.
- `A literature survey is required` → not market/user research.
- `Students learn hypothesis testing and randomized experiments` → direct experimental design; foundational or direct A/B testing depending on applied exercises.
- `Python is a prerequisite` → not Python training.
- `Students implement algorithms in Python` → direct Python training.
