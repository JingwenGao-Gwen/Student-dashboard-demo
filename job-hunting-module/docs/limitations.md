# Limitations

## 数据来源限制

- JD 来源于公开招聘网页，岗位内容会随时间变化。
- 当前仓库不包含完整原始 JD JSON，因此完整复现需要重新抓取公开网页。
- 招聘平台可能有访问频率限制、页面结构变化或反爬策略。

## 技能抽取限制

- 技能抽取聚焦“可以通过课程或训练提升的能力”，主动排除性格、态度和软性泛化描述。
- 同一技能可能存在中英文、缩写和近义写法，例如 SQL、数据库查询、数据提取。
- 技能频次不等于重要性，只表示在样本 JD 中被提及的强度。

## 课程推荐限制

- 课程推荐仅使用有 syllabus 的课程。
- 英文课程名来自学校课程列表，不手动翻译。
- AI 语义审核会显著减少关键词误判，但仍建议在正式面向学生发布前做抽样人工复核。
- 课程“基础支撑”不表示学生学完即可胜任岗位任务，而是说明课程提供必要的底层知识。

## 合规与发布建议

- 不建议公开上传完整 JD 正文、学校原始课程大纲、微信群聊天记录、图片和个人信息。
- 若仓库设为公开，建议只发布最终汇总表、方法说明、脚本和少量脱敏样例。
- 若需要完整复现和审计，应使用私有仓库存放原始中间数据。
